Chaitin's Ω: The Halting Probability

Lesson 1 ended with K(x) uncomputable. This lesson builds something stranger: a single, perfectly well-defined real number between 0 and 1 — one specific constant, not a family of them — whose digits secretly encode the true/false answer to the halting problem for every program that will ever be written. You could write down its first handful of digits with real effort. You could never, even in principle, write down all of them.

A prerequisite: self-delimiting programs

Why "self-delimiting" matters A set of binary strings is prefix-free if no string in the set is a prefix of another — you never need a delimiter to know where one code ends and the next begins, because reading bit by bit, the moment you've matched a valid code, you know you're done. Restrict attention to programs written in a prefix-free encoding (this is a real, standard restriction — not a cheat — and it costs nothing important, since any computable function can be encoded this way). Kraft's inequality then guarantees: Σ 2^(−|p|) ≤ 1, summed over any prefix-free set of codewords p. This single fact is what makes the rest of this lesson's sum well-defined instead of diverging to infinity.

The definition

Chaitin's Ω (Halting Probability) Ω = Σ 2^(−|p|), summed over every self-delimiting program p that halts.

Read it as a probability: generate a program by flipping a fair coin for each bit, one bit at a time, stopping the moment you've matched a valid self-delimiting codeword. Ω is exactly the probability that the program you build this way eventually halts. Because the programs summed over are prefix-free, Kraft's inequality guarantees 0 < Ω < 1 — a genuine, specific real number, not a divergent or ill-defined expression, despite being a sum over infinitely many halting programs of every possible length.

Why Ω answers the halting problem, one bit at a time

Here's the genuinely strange part. Suppose you somehow knew the first N bits of Ω exactly. Run every program of length up to N in parallel (dovetailing: run step 1 of every program, then step 2 of every program, and so on), and keep a running partial sum of 2^(−|p|) for every program p that has halted so far. This partial sum only ever grows, and it's always less than the true Ω (since it's missing contributions from programs that haven't halted yet, or haven't even been reached). Once the running partial sum's first N bits match your known first N bits of Ω, you can conclude with certainty: no other program of length ≤ N will ever halt — because if one did, eventually, it would push the partial sum's first N bits past what Ω actually allows, since Ω is the true total and your partial sum can never overshoot it. At that moment, you've settled the halting status — halts, or provably never halts — of every program up to length N, using nothing but those N bits.

The trade you're making You haven't cheated the halting problem's undecidability — you've paid for the answer up front, in the form of N bits of a number that (the next section shows) you can never actually obtain beyond a handful of them. Ω doesn't make the halting problem decidable; it relocates all of that problem's difficulty into a single, maximally compressed, and permanently inaccessible place.

Why Ω is uncomputable — and why its own digits are random

If a program could compute arbitrarily many digits of Ω on demand, the previous section shows you could feed it enough digits to decide the halting problem for programs of any length you like — which lesson 9 and lesson 15's diagonalization results already ruled out. So Ω itself must be uncomputable: no program outputs its digits, one after another, forever.

Ω is algorithmically random Beyond merely being uncomputable, Ω's first n bits are themselves algorithmically random in lesson 1's exact sense: K(Ω₁...Ωₙ) ≥ n − c. There is no shorter description of Ω's digits than the digits themselves — despite Ω being one single, specific, well-defined mathematical constant that you can characterize completely in one clean sentence ("the halting probability of a universal machine"). A short, precise definition of a number is not the same thing as a short description of its digits — Ω is the sharpest possible illustration of that gap.

In practice, researchers have computed the first handful of bits of Ω for specific toy machines (a handful of digits, with real effort) — but the theorem above guarantees this can never be extended into a general algorithm producing digit after digit indefinitely. Every bit past what's already been earned requires genuinely new, irreducible work; no shortcut, pattern, or formula can ever take over.

Practice before next lesson

Work a tiny toy version of Kraft's inequality by hand: pick three prefix-free binary codewords of your own choosing (e.g. lengths 1, 2, and 3 bits, chosen so none is a prefix of another — try 0, 10, 110). Compute Σ 2^(−|p|) for your three codewords and confirm it's ≤ 1. Then try to add a fourth codeword that breaks prefix-freeness (e.g. one that starts with an existing codeword) and see how the sum's meaning breaks down — the goal is to feel, on a tiny scale, why Ω's definition needs this property to be well-defined at all.
