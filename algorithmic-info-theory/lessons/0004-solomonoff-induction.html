Solomonoff Induction

Rationality's Bayes' theorem lesson gave you P(H|E) ∝ P(E|H)·P(H) and left one question quietly unanswered: where does the prior, P(H), actually come from before you've seen any evidence at all? This lesson gives Occam's razor — "prefer the simplest explanation" — an actual mathematical definition, using exactly the machinery from lessons 1–2, and turns it into a specific, principled choice of prior.

Algorithmic probability

Algorithmic Probability m(x) m(x) = Σ 2^(−|p|), summed over every self-delimiting program p that outputs x and halts. Compare directly to lesson 2's Ω, which sums over every halting program regardless of output — m(x) is the same construction, restricted to programs that produce this specific x. Interpreted the same way as Ω: the probability that a program built by flipping coins one bit at a time happens to output exactly x.

Because 2^(−|p|) shrinks exponentially with program length, m(x) is overwhelmingly dominated by x's shortest programs — the longer ones contribute almost nothing to the sum. This gives a genuinely deep connection, known as the coding theorem (Levin): −log₂ m(x) = K(x) + O(1). Algorithmic probability and Kolmogorov complexity are, up to a constant, the same information viewed two ways — a string with short description (low K) has high algorithmic probability; a string needing a long description (high K) has correspondingly tiny algorithmic probability.

Using m as a universal prior

Solomonoff Induction Given observed data, use m as the prior over hypotheses (each hypothesis being, formally, a program that could generate the data): P(hypothesis) ∝ m(hypothesis). Predict future observations by weighting every program consistent with what's been observed so far by its algorithmic probability, and combining their predictions for what comes next.

This makes Occam's razor an actual computation rather than a rule of thumb: since m assigns exponentially more weight to shorter programs, a simple hypothesis (a short program) that explains the data gets exponentially more prior weight than a needlessly convoluted one that explains it equally well. "Prefer simpler explanations, all else equal" isn't imported as an extra principle bolted onto Bayes' theorem — it falls straight out of using m as the prior, which is itself just lesson 1's K(x) restated as a probability distribution.

Why "universal" Solomonoff proved (a real theorem, not just a hopeful analogy) that induction using m as the prior converges to the true data-generating process about as fast as any method could, for any computable environment — a single prior that performs near-optimally no matter which computable process is actually generating the data, without needing to know which one in advance. This is the sense in which the method is "universal": it doesn't need to be told what kind of pattern to look for.

The catch — and a common overclaim to watch for

Solomonoff induction is uncomputable Computing m(x) requires summing over every halting program that outputs x — which requires knowing which programs halt, exactly the fact lesson 2 already proved is permanently out of reach. Solomonoff induction is a genuine mathematical ideal, an upper bound on how well inductive inference could possibly work — not an algorithm anyone runs.

Before accepting a claim like "modern AI systems approximate Solomonoff induction": ask precisely what's being approximated, and how. Real learning systems use specific, computable, practical methods (gradient descent over a fixed architecture, for instance) that were not derived from m(x) and don't compute anything resembling it. There may be a loose, qualitative family resemblance — both do something Occam's-razor-shaped, preferring simpler fits to data — but "approximates the uncomputable universal prior" is a much stronger, and generally unsupported, claim than "both happen to favor simplicity in some sense." Same discipline as every other misuse-check in this curriculum: ask exactly what's being computed, and compare it to the real, cited definition, not the evocative name.

Practice before next lesson

Take two strings of the same length: one highly compressible (like "ab" repeated 50 times) and one that looks incompressible to you (type random-seeming characters). Without computing anything exactly, reason through which one has higher algorithmic probability m(x) and why, using the coding theorem's relationship to K(x) from this lesson. Then write, in a sentence or two, what it would mean for a Solomonoff-induction-based predictor to have observed the first half of the compressible string and be asked to predict the rest — which hypotheses (programs) would dominate its prediction, and why.
