# Baseline: Programmer's Entry Point, Wants Abstraction for Its Own Sake

One of three new "modern, rigorous, not-widely-known" topics chosen the same session ([[../../quantum-mechanics/learning-records/0001-baseline.md|Quantum Mechanics]], [[../../algorithmic-info-theory/learning-records/0001-baseline.md|Algorithmic Information Theory]]). Strong Rust/C background, so type-system and function-composition analogies land immediately — no need to build up from abstract algebra prerequisites the way a pure-math course would.

**Implications:** Lead every new categorical concept with a programming example before (or instead of) a purely mathematical one. Move fast through definitions themselves (this user handles formal notation easily, per [[../../logic/learning-records/0001-baseline.md|Logic's baseline]]) and spend the saved time on worked examples showing the definition actually constrains something.
