Universal Properties & Products

Every construction so far — categories, functors, natural transformations — was defined by writing out concrete formulas. This lesson shows category theory's actual signature move: defining something purely by what it does in relation to everything else, never by what it's internally "made of." That style of definition is called a universal property, and once you see it applied to something as familiar as a pair of values, you'll see why category theorists reach for it constantly.

The product, defined the wrong way and the right way

The obvious way to define "the product of A and B": the set of ordered pairs {(a,b) : a∈A, b∈B}. That definition only makes sense in Set — pairs of what, in a preorder, or in your own type system's abstract objects? A universal property defines "product" once, for any category, using only objects and morphisms.

The product's universal property

Definition: Product Given objects A, B in a category, their product is an object P together with two morphisms π_A: P → A, π_B: P → B (the projections), such that: for any other object X with morphisms f: X → A and g: X → B, there exists a unique morphism ⟨f,g⟩: X → P making both triangles commute — π_A ∘ ⟨f,g⟩ = f and π_B ∘ ⟨f,g⟩ = g.

In Set, check this against the ordinary Cartesian product A×B with the usual projections π_A(a,b)=a, π_B(a,b)=b: given f:X→A and g:X→B, the map ⟨f,g⟩(x) = (f(x), g(x)) is exactly the unique function making both triangles commute — any other candidate function h:X→A×B satisfying π_A∘h=f and π_B∘h=g would have to send x to a pair whose first coordinate is f(x) and second is g(x), which is only one possible pair. Uniqueness isn't an extra assumption here — it falls straight out of what a pair is.

The same definition, a completely different answer

Apply the identical universal property inside lesson 1's preorder category (numbers under ≤, one morphism m→n exactly when m≤n). "Projections" π_A: P→A just mean P≤A; the universal property demands an object P with P≤m and P≤n, such that any X with X≤m and X≤n also satisfies X≤P — i.e., P is a lower bound for both m and n, and every other common lower bound is ≤ P. That's exactly the definition of P = min(m,n). (Uniqueness of the mediating morphism is automatic here for free: a preorder never has more than one morphism between any two objects.)

The payoff "Product" in Set means ordered pairs. "Product" in a number preorder means the minimum. Same universal property, wildly different concrete answers — because the universal property only specifies a relationship to every other object in the category, and lets each category's own structure determine what actually satisfies it. This is the real advantage over defining constructions case-by-case: prove a theorem using only the universal property, and it holds in Set, in preorders, and in every other category with products, all at once, with zero extra work per category.

A second, even simpler example: the terminal object

Definition: Terminal Object An object 1 such that for every object X in the category, there is a unique morphism X → 1.

In Set: any singleton set works (there's exactly one function from any set into a one-element set — send everything to the single element). In your own type system: the unit type (Rust's ()) — every type has exactly one function into it, the one that discards its input and returns (). Same universal-property pattern as products: one abstract specification, satisfied by a completely ordinary, already-familiar object in each category.

Why "the" product is justified — uniqueness up to isomorphism

Universal properties never pin down a construction to one specific object — they pin it down up to a unique isomorphism. Suppose both P and P′ satisfy the product universal property for the same A, B. Apply P′'s universal property to P's own projections (P counts as one of the "any object X with maps to A and B"): you get a unique φ: P → P′. Apply P's universal property to P′'s projections symmetrically: a unique ψ: P′ → P. Composing, ψ∘φ: P→P satisfies the same triangles as id_P — and by P's own universal property, the mediating morphism making those triangles commute is unique, so ψ∘φ = id_P. The same argument gives φ∘ψ = id_P′. So φ is an isomorphism — P and P′ aren't just similar, they're interchangeable, with a canonical (not arbitrarily chosen) isomorphism between them. This is why mathematicians comfortably say "the product," definite article, despite the universal property never naming one specific object.
