# Baseline: Experienced Developer, Interview Goal

User is an experienced developer who loves Rust and is comfortable with code. Goal is to pass technical interviews at Google-tier companies for a job change in the near future. No prior formal CS interview prep. Prefers Rust for all code examples, with C as fallback.

**Implications:** Skip basic programming concepts entirely — go straight to DS&A patterns and complexity analysis. Frame everything as "what would an interviewer actually test?" Use Rust idioms throughout.
