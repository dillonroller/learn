Truth Tables & Semantic Evaluation

Lesson 2 gave you a precise language for writing formulas. This lesson gives you a mechanical, purely algorithmic procedure for deciding whether any formula is true or false, and — critically — whether any argument is valid. No intuition required, no arguing about it: just enumerate every possibility and check.

Quick recall

From memory:

  • What's the main connective of (P ∧ Q) → R?
  • In P → Q, which is the antecedent?

Answers: → (it was applied last) · P is the antecedent, Q is the consequent

Why truth tables

Every atomic proposition is either true or false. A formula with n distinct atomic propositions has 2ⁿ possible combinations of truth values — and a truth table just lists every single one and evaluates the formula for each. This is exhaustive case analysis: brute-force enumeration, exactly like iterating i from 0 to 2ⁿ - 1 and reading each bit of i as a boolean.

The Rust translation Recall the Formula enum from lesson 2. A truth table is just this evaluator, called once per row: 
fn eval(f: &Formula, env: &HashMap<char, bool>) -> bool {
    match f {
        Formula::Atom(c)       => env[c],
        Formula::Not(a)        => !eval(a, env),
        Formula::And(a, b)     => eval(a, env) && eval(b, env),
        Formula::Or(a, b)      => eval(a, env) || eval(b, env),
        Formula::Implies(a, b) => !eval(a, env) || eval(b, env),
        Formula::Iff(a, b)      => eval(a, env) == eval(b, env),
    }
}
Generating the whole table is just looping over all 2ⁿ boolean assignments to the atoms and calling eval on each. Every table below was generated exactly this way — computed, not hand-typed — so there's no risk of a transcription slip.

Evaluating a compound formula

Tautology, contradiction, contingency

Every formula falls into exactly one of three categories, based on its final column:

  • Tautology — true in every row, no matter what. Structurally guaranteed true.
  • Contradiction — false in every row. Structurally guaranteed false.
  • Contingency — true in some rows, false in others. Depends on the actual facts.

Truth tables prove validity

Lesson 1 defined validity in prose: an argument is valid if the conclusion must be true whenever the premises are true. Truth tables make this fully mechanical:

The validity test An argument is valid exactly when there is no row where all the premises are true and the conclusion is false. Find one such row — a counterexample — and the argument is invalid, full stop. Check every row and find none, and the argument is valid, guaranteed.

Modus Ponens — valid

Premises: P, P → Q. Conclusion: Q.

Affirming the Consequent — invalid

Premises: P → Q, Q. Conclusion: P. This is the exact shape of lesson 1's dolphin argument ("all fish live in water, dolphins live in water, therefore dolphins are fish") — now we can prove formally why it fails.
