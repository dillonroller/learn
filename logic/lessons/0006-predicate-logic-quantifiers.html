Predicate Logic: Quantifiers & Predicates

Go all the way back to lesson 1's opening example — "all humans are mortal, Socrates is a human, therefore Socrates is mortal." Propositional logic can't actually formalize that argument. It has to treat "all humans are mortal" as one opaque atom — call it P — with no way to see that "human" and "mortal" are even related to "Socrates." Predicate logic is what fixes that: it opens up the inside of a sentence.

Quick recall

From lesson 2: P → Q is false in exactly one case — P true, Q false; everywhere else it's vacuously true. Keep that fact close, it resurfaces in a big way below.

Predicates: functions that return propositions

A predicate is a property or relation with a blank to fill in — not a proposition by itself, but something that becomes one once you plug in a specific thing. Write predicates as a name with parentheses: Man(x), Mortal(x), LivesInWater(x).

The Rust translation A predicate is exactly a function that returns bool: fn is_mortal(x: Thing) -> bool. The function itself isn't true or false — only a specific call like is_mortal(socrates) is. Same idea: Mortal(x) with a free variable x isn't a proposition yet; Mortal(Socrates) is — it's now a closed, evaluable statement, exactly like calling the function with a real argument.

The set of things you're allowed to plug in is called the domain of discourse — "all people," "all natural numbers," "all animals," whatever the conversation is actually about. Every quantified statement below is implicitly relative to some domain.

The universal quantifier: ∀

∀x P(x) reads "for all x, P(x)" — P holds for every single thing in the domain, no exceptions.

∀ pairs with →, not ∧ "All humans are mortal" symbolizes as ∀x (Human(x) → Mortal(x)) — "for anything, if it's human, then it's mortal." Not ∀x (Human(x) ∧ Mortal(x)), which would claim everything in the entire domain is both human and mortal — rocks, numbers, the domain itself, all of it. That's an absurd, almost certainly false claim. The conditional is what lets the statement talk about only the humans, while staying silent about everything else.

The existential quantifier: ∃

∃x P(x) reads "there exists an x such that P(x)" — at least one thing in the domain makes P true.

∃ pairs with ∧, not → "Some birds cannot fly" symbolizes as ∃x (Bird(x) ∧ ¬Fly(x)) — "there's something that is a bird and can't fly." Not ∃x (Bird(x) → ¬Fly(x)). Remember the vacuous-truth fact from the recall above: a conditional is true whenever its antecedent is false. So Bird(x) → ¬Fly(x) is satisfied by any x that isn't a bird at all — a rock, a number, anything non-bird makes the conditional vacuously true. That formula would be satisfied even in a world with zero non-flying birds, as long as one non-bird exists anywhere in the domain. It says almost nothing.
The pairing rule, side by side ∀x (P(x) → Q(x)) — "everything that's P is also Q."
∃x (P(x) ∧ Q(x)) — "something is both P and Q."
Swap the connectives and you get either an absurdly strong claim (∀ + ∧) or an almost-empty one (∃ + →). This is the single most common mistake when learning predicate logic — and it's really lesson 2's necessary/sufficient distinction showing up again in a new notation.

Negating quantifiers

Quantifiers have their own De Morgan-style duality — pushing a negation through a quantifier flips which quantifier it is:

OriginalEquivalent
¬∀x P(x)∃x ¬P(x)
¬∃x P(x)∀x ¬P(x)
Direct extension of lesson 4's De Morgan laws "Not everyone likes pizza" (¬∀x Likes(x)) means exactly "someone doesn't like pizza" (∃x ¬Likes(x)). "No one is perfect" is ¬∃x Perfect(x), which is the same claim as ∀x ¬Perfect(x) — "everyone is imperfect." Same shape as ¬(P∧Q) ≡ ¬P∨¬Q: negation flips the quantifier the same way it flips the connective.
