Symbolizing English Arguments in FOL

Lesson 6 gave you one predicate and one quantifier at a time. Real arguments rarely stay that simple — they involve relationships between multiple things, and multiple quantifiers stacked on top of each other. This lesson covers the two skills that make that manageable: predicates with more than one slot, and getting quantifier order right — which turns out to matter enormously.

Quick recall

From lesson 6: ∀ pairs with →, ∃ pairs with ∧. Getting this backwards produces either an absurdly strong claim or an almost-vacuous one.

Multi-place predicates: relations

So far every predicate has taken one slot: Mortal(x). A relation is a predicate with two or more slots, describing how things relate to each other: Loves(x, y), Taller(x, y), Between(x, y, z).

The Rust translation Same idea as lesson 6's predicate-as-function analogy, just more arguments: fn loves(x: Person, y: Person) -> bool. Loves(x, y) isn't the same claim as Loves(y, x) — argument order matters exactly the way it does in a function call. "Alice loves Bob" is Loves(Alice, Bob), and swapping the arguments changes the meaning entirely.

Quantifier order matters — a lot

This is the single biggest trap in predicate logic. Compare these two formulas, which use the exact same predicate and the exact same two quantifiers, in a different order:

FormulaEnglish
∀x ∃y Loves(x, y)"Everyone loves someone" — each person has somebody they love, possibly a different somebody for each person
∃y ∀x Loves(x, y)"There's someone whom everyone loves" — one specific person, loved by literally everybody

These are wildly different claims, and the second is much stronger. Here's a concrete scenario that makes the difference undeniable — a "love triangle" among three people, Alice, Bob, and Carol:

xloves y
AliceBob
BobCarol
CarolAlice

Check ∀x ∃y Loves(x, y): does Alice love someone? Yes, Bob. Does Bob love someone? Yes, Carol. Does Carol love someone? Yes, Alice. True — everyone has somebody.

Now check ∃y ∀x Loves(x, y): is there one single person loved by all three? Try y = Bob: does Bob love Bob? No (Bob loves Carol) — fails. Try y = Carol: does Alice love Carol? No (Alice loves Bob) — fails. Try y = Alice: does Bob love Alice? No — fails. False — no single person is loved by everyone.

Same predicate, same quantifiers, opposite truth values In this exact scenario, ∀x∃y Loves(x,y) is true and ∃y∀x Loves(x,y) is false — proof by concrete example that swapping quantifier order changes the claim. As a rule of thumb: ∃ placed before a ∀ is claiming one fixed thing works for everyone; ∃ placed after a ∀ allows a different thing for each case. Read left to right, and don't casually swap the order when translating.

Common English patterns

EnglishFOLNote
"Everyone who is X is Y"∀x (X(x) → Y(x))same ∀/→ pairing, just with "who" spelled out
"Someone who is X is Y"∃x (X(x) ∧ Y(x))same ∃/∧ pairing
"Only X are Y"∀x (Y(x) → X(x))necessary condition — reversed, like lesson 2's "only if"
"No X is Y"∀x (X(x) → ¬Y(x))from lesson 6 — the negated consequent
"Only" strikes again "Only citizens can vote" doesn't mean "if you're a citizen, you can vote" (that's backwards — plenty of citizens can't vote, e.g. minors). It means voting requires citizenship: ∀x (CanVote(x) → Citizen(x)). This is exactly the necessary-vs-sufficient trap from lesson 2, wearing a quantifier now instead of a bare conditional.

Full circle: symbolizing the Socrates argument

Back in lesson 1, "all humans are mortal, Socrates is a human, therefore Socrates is mortal" could only be gestured at in prose. Now it's precise:

The Socrates argument, fully formalized Premise 1: ∀x (Human(x) → Mortal(x))
Premise 2: Human(Socrates)
Conclusion: Mortal(Socrates)

Notice Socrates is an individual constant, not a variable — it refers to one specific, fixed thing in the domain directly. Contrast this with x in Human(x): x is a placeholder that only gets meaning once a quantifier binds it (∀x or ∃x); Socrates needs no quantifier because it already picks out exactly one thing on its own. We have everything needed to state this argument with total precision. Proving it formally — showing step by step why Mortal(Socrates) really does follow — is exactly what lesson 8 does.
