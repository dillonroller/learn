Soundness & Completeness

You now have two completely different ways to establish that an argument works. Lesson 3 checks every possible truth-value assignment — a purely semantic method, about what's true. Lesson 5 chains inference rules into a proof — a purely syntactic method, about what's derivable. This lesson asks the question that's been quietly sitting underneath both: do these two methods always agree with each other? The answer has a name — actually two names — and they're the whole reason you can trust the rules you've been using.

Quick recall

From memory:

  • What does ⊢ mean?
  • What does it mean for an argument to be valid, by the truth-table method?

Answers: ⊢ means "derivable from the premises using the proof rules" · valid means no row of the truth table makes the premises true and the conclusion false

Two symbols for two different questions

Lesson 5 gave ⊢ its own name — the turnstile, read "proves" or "derives." Validity, as lesson 3 defined it, deserves a matching symbol: ⊨, read "entails" or "models."

SymbolQuestion it answersMethod
Premises ⊢ ConclusionCan you build a proof, one licensed step at a time?Syntactic — apply inference rules (lesson 5)
Premises ⊨ ConclusionIs there any way to make the premises true and the conclusion false?Semantic — check every truth-value assignment (lesson 3)
⊢ and ⊨ are two different tools aimed at the same target Both symbols are trying to answer "does the conclusion really follow?" — but ⊢ answers it by building something (a proof), while ⊨ answers it by checking everything (every row of a truth table). They could, in principle, disagree: a system's rules might be too weak to prove something that's actually valid, or — worse — too permissive, letting you prove something that isn't actually valid at all. Whether they agree is exactly what this lesson is about.

Soundness: the rules don't lie to you

A proof system is sound when everything it can prove is actually true-preserving:

Soundness (of a system) If Premises ⊢ Conclusion, then Premises ⊨ Conclusion. In words: whenever a proof exists, the argument really is valid. A sound system never lets you derive a semantically invalid argument — the rules can't be tricked into manufacturing a false conclusion from true premises.
Wait — didn't lesson 1 already define "sound"? Yes, and this is a genuinely different use of the same English word, not a special case of it. Lesson 1's soundness is a property of one specific argument: valid structure plus true premises. This soundness is a property of an entire proof system — the whole rulebook of MP, MT, HS, DS, and friends. One is about whether a particular argument's premises happen to be true; the other is about whether the rulebook itself can ever be fooled. Logicians overload the word constantly — context is what disambiguates "a sound argument" from "a sound proof system," and now you have both senses on file.

Completeness: the rules are powerful enough

Soundness only protects you in one direction. The other direction has its own name:

Completeness (of a system) If Premises ⊨ Conclusion, then Premises ⊢ Conclusion. In words: whenever an argument really is valid, a proof exists using the rules. A complete system never leaves a truly valid argument unprovable — nothing true slips through the cracks.
Put together: ⊢ and ⊨ coincide exactly Soundness gives you ⊢ → ⊨. Completeness gives you ⊨ → ⊢. A system with both gives you ⊢ if and only if ⊨ — provability and validity become two different roads to the exact same destination. This is not automatic for every conceivable rulebook; it has to be proven for a specific system.

The system from lessons 5 and 8 has both

The natural deduction system you've been using — Modus Ponens, Modus Tollens, Universal Instantiation, Existential Generalization, and the rest — is not just a convenient toolkit. It was proven, rigorously, to be both sound and complete for first-order logic. That proof is Kurt Gödel's completeness theorem, from 1929 — two years before the far more famous result that gets his name attached to a very different kind of theorem, which is exactly where lesson 11 is headed. Don't let the shared name merge them in your head; they answer different questions, and this lesson's result is the reassuring one.

What this buys you, practically Because the system is sound: every proof you build in lessons 5 and 8 is a guarantee, not a suggestion — if you followed the rules correctly, the conclusion really is warranted by the premises. Because the system is complete: whenever an argument is genuinely valid, a proof exists, full stop — completeness is an existence guarantee, not a "here's how to find it" guarantee. If you're stuck constructing a proof for something you're confident is valid, the problem is your search, not a hole in the rulebook.

A related but separate question: decidability

Completeness says a proof exists. It says nothing about how long it takes to find one, or whether you can always even tell.

Decidable vs. complete — not the same thing Propositional logic is decidable: the truth-table method from lesson 3 is a mechanical procedure that is guaranteed to terminate — for n atoms, exactly 2ⁿ rows, then you're done, valid or not. Full first-order logic is complete but not decidable: there is no general algorithm guaranteed to always terminate and tell you, for an arbitrary FOL argument, whether it's valid or not (a result due to Church and Turing, 1936). If the argument is valid, completeness guarantees a proof exists and a systematic search will eventually find it. If it's not valid, that same search may simply never terminate — there's no guaranteed way to know you should give up.

Practice before next lesson

Go back to one of the proofs you built or read in lesson 5 or lesson 8. That proof, by itself, is direct evidence that ⊢ holds. Now notice: because the system is sound, you get ⊨ for free — you already know the argument is valid without building a single truth-table row. Soundness is what lets a proof stand in for a truth table.
