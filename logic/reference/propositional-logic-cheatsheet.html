Arguments

TermDefinitionTest
ValidConclusion must be true if premises areNo row: premises true, conclusion false
SoundValid + all premises actually trueValid, and check the facts
DeductivePremises guarantee the conclusionCertainty, if valid + sound
InductivePremises make conclusion probableEvaluated by strength, not validity

Validity is about structure, not truth. A valid argument can have a false premise and false conclusion. Only sound arguments guarantee a true conclusion.

The five connectives

SymbolNameRust / C
¬PNegation — "not P"!p
P ∧ QConjunction — "P and Q"p && q
P ∨ QDisjunction — "P or Q" (inclusive)p || q
P → QConditional — "if P then Q"!p || q
P ↔ QBiconditional — "P iff Q"p == q

In P → Q: P is the antecedent, Q is the consequent. P → Q is false in exactly one case — P true, Q false. Every other case is true (including vacuously, whenever P is false).

Translating English

EnglishSymbolizes asNote
"P unless Q"P ∨ Qunless ≈ or
"if P, then Q"P → Qplain form
"P only if Q"P → Qsame as "if P then Q"
"P if Q"Q → Preversed — no "only"
"neither P nor Q"¬P ∧ ¬Qboth false
Necessary vs. sufficient — the trap "Only if" states a necessary condition (required, but maybe not enough on its own): "pass only if you study" = Pass → Study. Plain "if" states a sufficient condition (enough on its own): "pass if you study" = Study → Pass. Dropping the word "only" reverses the arrow.

The falsification test (always works): find the one scenario that would make the English sentence a lie, then match it to the one scenario that makes P → Q false (P true, Q false).

Watch modal words — "may," "can," "must," "should" change what the atomic proposition claims. "You may enter" is about permission, not the physical act of entering. Pin down exactly what each proposition means before symbolizing.

Well-formed formulas

φ ::= P | Q | R | ...
    | ¬φ
    | (φ ∧ φ)
    | (φ ∨ φ)
    | (φ → φ)
    | (φ ↔ φ)

Recursive grammar — same shape as a Rust enum with Box for recursion. The main connective is whichever was applied last; it determines the formula's overall shape.

Truth tables

Rows needed: 2ⁿ for n atomic propositions.

Tautology: true in every row.

Contradiction: false in every row.

Contingency: true in some rows, false in others.

Validity test: valid iff no row makes all premises true and the conclusion false. One counterexample row disproves validity entirely.

Equivalence test: two formulas are logically equivalent iff their truth tables match on every row (e.g. P → Q ≡ ¬P ∨ Q).

Normal forms

CNF: AND of ORs — (l₁∨l₂)∧(l₃∨l₄)∧.... Required input format for SAT solvers.

DNF: OR of ANDs — (l₁∧l₂)∨(l₃∧l₄)∨.... Mirror of CNF.

Conversion recipe: eliminate → (Material Implication), push ¬ inward (De Morgan + Double Negation), then distribute to the target shape.
