Common Distributions

Lesson 1 gave you the machinery (spaces, random variables). Lesson 2 gave you the tools to summarize a distribution (E, Var). This lesson applies both to the handful of distributions that show up so often they get their own names — and along the way, introduces a genuine subtlety: continuous random variables break one thing you've taken for granted since lesson 1.

Bernoulli & Binomial — counting successes

Bernoulli(p) and Binomial(n, p) A Bernoulli(p) random variable is a single coin flip with bias p: P(X=1) = p, P(X=0) = 1−p. A Binomial(n, p) random variable is the count of successes in n independent Bernoulli(p) trials: P(X=k) = C(n,k) pᵏ(1−p)ⁿ⁻ᵏ.

Binomial's PMF looks like it demands real work to find E[X]. It doesn't — because a Binomial(n,p) is secretly just X₁ + X₂ + ... + Xₙ, a sum of n Bernoulli(p) variables. By lesson 2's linearity of expectation: E[X] = n · E[X₁] = n · p — no sum over k needed, ever. The same shortcut that solved the card-shuffle problem solves this one for free.

A genuinely new fact: variance needs independence Linearity of expectation held unconditionally — dependent or not. Variance doesn't get that for free: Var(X+Y) = Var(X) + Var(Y) only when X and Y are independent (in general there's a correlation term). The n Bernoulli trials making up a Binomial genuinely are independent by construction, so Var(X) = n · Var(X₁) = n · p(1−p) — but this is a real, separate fact from linearity, not the same shortcut in disguise. Keep the asymmetry straight: expectation always adds; variance only adds under independence.

Poisson — counting rare events

Poisson(λ) Models the count of rare, independent events over a fixed window (time, area, etc.), with average rate λ: P(X=k) = e⁻λ λᵏ / k!. E[X] = Var(X) = λ.

Poisson isn't an arbitrary formula — it's what Binomial(n,p) becomes in the limit of many trials, each individually very unlikely: let n → ∞ and p → 0 while holding np = λ fixed (more and more chances for something rare to happen, each chance individually tinier). Concretely: modeling "typos per page" as Binomial would require picking an arbitrary number of "trials" (characters? words?); Poisson sidesteps that by only needing the average rate λ, which is exactly the situation you're usually actually in.

The Normal distribution — and a subtlety continuous variables force on you

Bernoulli, Binomial, and Poisson are all discrete — countable possible values, each with its own positive probability. The Normal (Gaussian) distribution is continuous — its values range over an entire interval of real numbers, and that changes the rules.

Normal(μ, σ²) Probability density function: f(x) = (1 / σ√(2π)) · e^(−(x−μ)²/2σ²). E[X] = μ, Var(X) = σ². Probabilities come from area under the curve: P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx.
The subtlety: P(X = x) = 0 for every exact value For a continuous random variable, the probability of landing on any single, exact value is always zero — not "very small," genuinely zero. A person's height is never exactly 5.83920... feet with positive probability; only intervals ("between 5.8 and 5.9 feet") have positive area under the curve and therefore positive probability. This isn't a paradox, but it does mean lesson 1's PMF (p(x) = P(X=x)) is literally identically zero for a continuous X and useless as a description — which is exactly why continuous variables need a density (something you integrate) instead of a mass (something you sum), and why this lesson waited until now to introduce one properly.

The famous "68–95–99.7 rule": about 68% of a Normal distribution's probability sits within one standard deviation of the mean, 95% within two, 99.7% within three — worth memorizing as a sanity check, though the exact numbers come from integrating the density above.

Check your understanding

1. Why can E[Binomial(n,p)] be computed as n·p without summing the PMF over all k?

2. Why does Var(Binomial) = n·p(1−p) require an extra assumption that E[Binomial] = n·p did not?

3. In what sense is the Poisson distribution a limit of the Binomial?

4. Why is P(X = 5.0 exactly) = 0 for a continuous random variable, even though 5.0 is a value X could plausibly take?

Practice before next lesson

Compute E[X] and Var(X) for X ~ Binomial(20, 0.25) two ways: first using the shortcut (n·p and n·p(1−p)), then — for just k=0,1,2 — compute a few actual PMF values C(20,k)(0.25)ᵏ(0.75)²⁰⁻ᵏ by hand to get a feel for how much harder the direct route would be for the full expectation sum. Then sketch (roughly, by hand or in words) what a Normal(0,1) density looks like, and shade the region corresponding to P(−1 ≤ X ≤ 1) — connecting the abstract 68% figure to an actual picture.
